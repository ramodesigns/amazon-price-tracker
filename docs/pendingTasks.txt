=== SECTION 9: PRE-PROD TESTING SEQUENCE ===
================================================================================
# PRIORITY: HIGH
# ACTION: Follow this sequence when testing in pre-prod environment
# NOTE: This is a test script to validate all functionality works

STEP 1: Plugin Activation
  - Activate plugin via WordPress admin
  - Verify: wp_apt_products table exists
  - Verify: wp_apt_price_history table exists
  - Verify: wp_apt_user_settings table exists
  - Verify: wp_apt_blacklist table exists
  - Verify: apt_encryption_key option exists
  - Verify: Cron job apt_scheduled_price_refresh is scheduled

STEP 2: Configure User Settings
  - Endpoint: PUT /wp-json/amazon-price-tracker/v1/settings
  - Body: {"access_key": "YOUR_KEY", "secret_key": "YOUR_SECRET", "partner_tags": {"US": "your-tag-20"}}
  - Expected: 201 Created (first time) or 200 OK (update)

STEP 3: Validate Amazon Credentials
  - Endpoint: POST /wp-json/amazon-price-tracker/v1/settings/validate
  - Expected: {"valid": true, "message": "..."}

STEP 4: Create First Product
  - Endpoint: POST /wp-json/amazon-price-tracker/v1/products
  - Body: {"asin": "B08N5WRWNW", "region": "US"}
  - Expected: 201 Created with product object

STEP 5: Verify Product Data
  - Endpoint: GET /wp-json/amazon-price-tracker/v1/products/{id}
  - Verify: images array populated
  - Verify: facts object contains title, brand, etc.

STEP 6: Test Price Refresh
  - Endpoint: POST /wp-json/amazon-price-tracker/v1/products/{id}/refresh
  - Expected: 200 OK with updated product
  - Verify: New record in apt_price_history

STEP 7: Test Price History
  - Endpoint: GET /wp-json/amazon-price-tracker/v1/products/{id}/prices
  - Verify: Returns price records
  - Test with ?aggregate=daily

STEP 8: Test Blacklist
  - Endpoint: POST /wp-json/amazon-price-tracker/v1/blacklist
  - Body: {"asin": "B000000000", "region": "US", "reason": "Test"}
  - Verify: 201 Created
  - Try to create product with blacklisted ASIN
  - Verify: 403 Forbidden with BLACKLISTED code

STEP 9: Test Rate Limiting (as non-admin user)
  - Create standard WordPress user (Subscriber role)
  - Generate Application Password for user
  - Create 50 products successfully
  - Attempt 51st product
  - Verify: 429 Rate Limit Exceeded

STEP 10: Test Scheduled Refresh
  - Go to Settings > Amazon Price Tracker
  - Click "Run Manual Refresh"
  - Verify: Success message with count
  - Check price_history table for new records
