Amazon Price Tracker API (1.0.0)

Download OpenAPI specification:

A WordPress REST API plugin for tracking Amazon product prices across multiple international marketplaces. Integrates with Amazon's Creators API to fetch product data and pricing, storing historical price records over time.

Authentication

All endpoints except GET /health require authentication. This API relies entirely on WordPress's own authentication layer - there is no bespoke API-key system. The two supported mechanisms are:

  • Application Passwords (HTTP Basic Auth) - the practical choice for external REST clients (Insomnia, Postman, curl, server-to-server integrations). Generate one under the target WordPress user's profile ("Application Passwords" section), then send it as Authorization: Basic base64(username:application_password). Requires either HTTPS or WP_ENVIRONMENT_TYPE set to local (WordPress disables Application Passwords over plain HTTP otherwise, except for local environments).
  • Cookie + nonce auth - used implicitly by same-origin JavaScript running inside wp-admin (e.g. wp.apiFetch). Not practical for external clients and not modeled separately below; treat basicAuth as the representative scheme.

Endpoints marked "Admin only" additionally require the authenticated user to have the manage_options WordPress capability (i.e. be an Administrator).

Error shape

Non-2xx responses always follow WordPress's standard WP_Error REST serialization:

{ "code": "ASIN_NOT_FOUND", "message": "Product not found on Amazon", "data": { "status": 400 } }

data.status mirrors the HTTP status code. Some error codes add extra fields inside data (e.g. data.errors for validation failures, data.limit/data.used/data.resets_at for rate limiting) - see the ErrorResponse variants under each operation.

Rate limiting

Non-admin users are capped at a configurable number of product creations per day (default 50, via the apt_daily_creation_limit WordPress option). POST /products and POST /products/bulk add X-RateLimit-Limit, X-RateLimit-Remaining, and X-RateLimit-Reset response headers for non-admin callers. Admins are exempt from the limit entirely (no headers, no 429s).

Reactivate a previously soft-deleted product

Brings a soft-deleted (is_active: false) product back, re-fetching current data from Amazon Creators API rather than reusing whatever was last stored, and recording a fresh price history entry. The existing row's id, created_at, and created_by are preserved - this updates the existing row, it does not create a new one.

A deliberately separate endpoint from POST /products (rather than having creation silently revive a soft-deleted row) so reactivation is a distinct, auditable action rather than an implicit side effect of create.

Subject to the same blacklist check as POST /products. Also subject to the same non-admin daily rate limit as a gate (blocked if you're already at today's cap) - but note it does not itself count against that cap, since it updates an existing row rather than inserting a new one, and the rate limit is derived from counting rows created that day.

Duplicate ASIN/region pairs within the request are detected and skipped before any Amazon calls are made; everything else (rate limiting, blacklist, existing-product conflicts) is evaluated per item exactly as it would be for a single POST /products call.
